# نوت‌پد آنلاین با Vanilla JS + Supabase + Vercel

این نسخه برای تست ساده با **یوزرنیم + رمز عبور عددی** ساخته شده و دیگر از Supabase Auth ایمیلی استفاده نمی‌کند. بنابراین خطای زیر دیگر اتفاق نمی‌افتد:

```txt
email rate limit exceeded
```

## چرا نسخه جدید؟

در نسخه قبلی، چون پشت‌صحنه برای هر یوزرنیم یک ایمیل داخلی ساخته می‌شد، Supabase Auth ممکن بود هنگام ثبت‌نام ایمیل تأیید ارسال کند. سرویس ایمیل داخلی Supabase برای تست محدودیت دارد و ممکن است خطای rate limit بدهد.

در این نسخه، احراز هویت با تابع‌های PostgreSQL/RPC انجام می‌شود:

```txt
app_register
app_login
app_logout
app_list_notes
app_create_note
app_update_note
app_delete_note
```

یعنی پروژه هیچ ایمیلی ارسال نمی‌کند.

## امکانات

- ورود و ثبت‌نام با یوزرنیم
- رمز عبور عددی با حداقل ۴ رقم؛ مثل `1234`
- ذخیره نوت‌ها در Supabase
- جداسازی نوت‌ها برای هر کاربر با session token سفارشی
- ساخت، ویرایش، حذف و پین کردن نوت
- جستجو در عنوان و متن
- طراحی RTL و مناسب فارسی
- آماده Deploy روی Vercel

## نکته امنیتی مهم

این روش برای پروژه تستی، دانشگاهی یا نمونه‌کار سبک مناسب است. برای پروژه واقعی و عمومی بهتر است از Supabase Auth رسمی با ایمیل/OTP/Google یا یک بک‌اند امن استفاده کنی.

رمزها در دیتابیس به‌صورت مستقیم ذخیره نمی‌شوند و با `pgcrypto` هش می‌شوند، اما چون کل منطق ورود از فرانت صدا زده می‌شود، همچنان سطح امنیت آن مثل یک سیستم Auth کامل تولیدی نیست.

## مرحله ۱: اجرای SQL جدید

داخل Supabase برو به:

```txt
SQL Editor > New query
```

کل محتوای فایل `supabase.sql` را اجرا کن.

این فایل جدول‌ها و تابع‌های جدید زیر را می‌سازد:

```txt
app_users
app_sessions
app_notes
```

نکته: اگر نسخه‌های قبلی جدول `notes` یا `profiles` ساخته باشند، این نسخه کاری با آنها ندارد. نسخه جدید از جدول‌های `app_*` استفاده می‌کند.

## مرحله ۲: تنظیمات پروژه

آدرس پروژه فعلی:

## خطاهای رایج

### email rate limit exceeded

این خطا مربوط به نسخه قدیمی بود که از Supabase Auth ایمیلی استفاده می‌کرد. نسخه جدید ایمیل ارسال نمی‌کند. فایل‌های پروژه را جایگزین کن، `supabase.sql` جدید را اجرا کن و دوباره Deploy بگیر.

### تابع‌های دیتابیس ساخته نشده‌اند

اگر خطایی مثل `function app_login does not exist` دیدی، یعنی فایل `supabase.sql` جدید هنوز اجرا نشده است.

### یوزرنیم یا رمز اشتباه است

یا یوزرنیم ثبت نشده، یا رمز اشتباه وارد شده است. برای تست، یک یوزرنیم جدید با رمز `1234` بساز.

### صفحه مستقیم با file:// باز نمی‌شود

چون پروژه از ES Module استفاده می‌کند، باید با یک local server اجرا شود.


## رفع خطای تابع‌ها یا pgcrypto

اگر پیام «تابع‌های دیتابیس ساخته نشده‌اند» یا خطای مربوط به `crypt` / `gen_salt` دیدی، فایل `supabase.sql` همین نسخه را دوباره کامل در SQL Editor اجرا کن.

در این نسخه `pgcrypto` داخل schema `extensions` فعال می‌شود و تابع‌های دیتابیس با `search_path = public, extensions` ساخته می‌شوند؛ این مورد برای Supabase لازم است، چون اکستنشن‌ها معمولاً در schema جداگانه `extensions` هستند.

بعد از اجرای SQL، انتهای خروجی باید لیست تابع‌های `app_*` را نشان بدهد. اگر خروجی خالی بود یعنی SQL کامل اجرا نشده است.

## نسخه طراحی مینیمال بدون بک‌گراند تصویری

در این نسخه، طراحی صفحه ورود و صفحه افزودن/ویرایش یادداشت بازطراحی شده است:

- بک‌گراند کاملاً ساده است و هیچ عکس یا تصویر پس‌زمینه‌ای استفاده نشده است.
- تم روشن، مینیمال و گرم با رنگ‌های کرم، قهوه‌ای و خاکی پیاده‌سازی شده است.
- صفحه ورود به یک کارت مرکزی ساده تبدیل شده است.
- صفحه اصلی شامل سایدبار راست، جستجو، لیست یادداشت‌ها و ادیتور بزرگ سمت چپ است.
- منطق دیتابیس، RPC و Supabase تغییر نکرده تا خطای جدید ایجاد نشود.

## عیب‌یابی سریع بات تلگرام

اگر Webhook ست شد ولی بات جواب نمی‌دهد، اول آدرس زیر را در مرورگر باز کن:

```txt
https://YOUR-VERCEL-DOMAIN.vercel.app/api/telegram
```

باید `ok: true` و وضعیت Environment Variableها را ببینی. اگر هرکدام `false` بود، همان متغیر را در Vercel اضافه کن و Redeploy بگیر.

بعد در Vercel از بخش `Functions / Logs` خطاهای `/api/telegram` را ببین. نسخه جدید اگر خطای Supabase یا Telegram رخ بدهد، تلاش می‌کند متن خطا را داخل خود چت تلگرام هم بفرستد.

## تغییرات نسخه v11 - Telegram Buttons + Edit + Web App

این نسخه بات تلگرام را کامل‌تر می‌کند:

- دکمه «ویرایش متن» برای هر یادداشت اضافه شد.
- کاربر می‌تواند متن یادداشت را داخل تلگرام ویرایش کند.
- اگر متن جدید به شکل `عنوان جدید | متن جدید` ارسال شود، عنوان و متن هر دو تغییر می‌کنند.
- دکمه `🌐 باز کردن وب‌اپ` به منوی بات اضافه شد.
- فایل `api/telegram-setup.js` اضافه شد تا Command Menu کنار تکست‌باکس تلگرام تنظیم شود.
- وب‌اپ با Telegram Web App SDK سازگارتر شد و داخل تلگرام expand/ready می‌شود.

برای اینکه دکمه وب‌اپ داخل تلگرام کار کند، این Env را داخل Vercel اضافه کن:

```env
PUBLIC_APP_URL=https://YOUR-VERCEL-DOMAIN.vercel.app
```

بعد از Deploy، برای ست‌کردن commands کنار تکست‌باکس، این URL را باز کن:

```txt
https://YOUR-VERCEL-DOMAIN.vercel.app/api/telegram-setup?secret=TELEGRAM_WEBHOOK_SECRET
```

اگر می‌خواهی دکمه پایین تلگرام به‌جای Command Menu مستقیماً وب‌اپ را باز کند:

```txt
https://YOUR-VERCEL-DOMAIN.vercel.app/api/telegram-setup?secret=TELEGRAM_WEBHOOK_SECRET&menu=webapp
```

برای راه‌اندازی کامل، فایل زیر را بخوان:

```txt
TELEGRAM_SETUP.md
```

### SQL / DAX / Python

برای این سه زبان، هایلایتینگ فعال است. چون Prettier به‌صورت رسمی formatter داخلی برای SQL، DAX و Python ندارد، دکمه فرمت در این پروژه برای آن‌ها از formatter داخلی سبک استفاده می‌کند:

- SQL: خواناتر کردن keywordها، joinها، where/group/order و خطوط شرطی
- DAX: خواناتر کردن VAR / RETURN و جداکردن آرگومان‌ها
- Python: مرتب‌سازی فاصله‌ها، تبدیل tab به ۴ فاصله و indentation پایه

### نکته مهم دیتابیس

بعد از جایگزینی این نسخه، فایل `supabase.sql` را یک‌بار در Supabase اجرا کن، چون ستون `language` و امضای جدید RPCهای `app_create_note` و `app_update_note` اضافه شده‌اند.


## رفع خطای تغییر return type تابع‌ها

اگر هنگام اجرای SQL خطای زیر دیدی:

```txt
ERROR: 42P13: cannot change return type of existing function
HINT: Use DROP FUNCTION app_list_notes(uuid) first.
```

این نسخه اصلاح شده و ابتدای فایل `supabase.sql` قبل از ساخت RPCهای جدید، تابع‌های قدیمی را `DROP FUNCTION IF EXISTS` می‌کند. کافی است همین `supabase.sql` نسخه v12.1 را کامل اجرا کنی.
